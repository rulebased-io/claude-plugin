# eval-log 실행 타이밍 개선

- **생성일**: 2026-03-15
- **상태**: backlog
- **우선순위**: medium

## 내용

eval-log가 대화 하나 끝날 때마다 실행되면 너무 잦음.
커밋 시점이나 큰 작업 그룹이 마무리될 때 실행되는 게 적절함.

## 트리거 후보

- git commit hook (post-commit)에서 실행
- 특정 커밋 수 이상 쌓였을 때
- 사용자가 수동으로 실행
- Stop hook에서 조건부 실행 (예: 세션 길이가 일정 이상일 때만)

## 판단 포인트

- post-commit hook이 가장 자연스러운 타이밍
- 매 커밋마다는 과할 수 있으니 조건부 (예: 세션에서 5개 이상 커밋 후)
- Hook의 Stop 이벤트보다 git hook이 더 적절할 수 있음
