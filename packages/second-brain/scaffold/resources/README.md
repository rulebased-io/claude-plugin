# Resources

Topics of interest and reference materials. This is where most knowledge lives.

## What goes here

- **Learning notes** — summaries, key takeaways from books/articles/videos
- **Code snippets** — reusable patterns, commands, configurations
- **TIL (Today I Learned)** — short daily learning entries
- **Reference docs** — frequently consulted information
- **Collected knowledge** — evergreen notes on topics you care about

## Organization

Use **tags** and **links** as the primary classification mechanism:

```yaml
tags: [reading, distributed-systems]    # a reading note
tags: [snippet, python, async]          # a code snippet
tags: [til, git]                        # a TIL entry
```

Start flat. When a topic naturally accumulates enough notes, create a subfolder to group them. See `system/growth-triggers.md` for specific thresholds.

## Maturity

- `status: seedling` — rough note, just captured
- `status: budding` — refined, has connections
- `status: evergreen` — mature, densely linked, trustworthy
