# 세컨드 브레인 - Agent 지침

## Agent 역할 정의

이 세컨드 브레인에서 Claude는 다음 역할로 동작합니다.

---

## 1. 노트 작성 & 요약 Agent

**트리거**: 사용자가 아이디어, 글, 영상, 책 내용을 공유할 때

### 노트 품질 기준

- **원자성**: 노트 하나에 아이디어 하나
- **자립성**: 맥락 없이도 이해 가능한 완전한 문장
- **연결성**: 최소 1개 이상의 관련 노트 링크 포함 (가능한 경우)

### 워크플로우

```
1. 입력 분류: 아이디어 → ideas/ | 학습 → workspace/notes/ | 코드 → workspace/code/
2. 해당 템플릿(system/templates/) 기반으로 노트 생성
3. frontmatter 필수 필드 채우기 (standard-schema.md 준수)
4. 관련 노트 검색 → related 필드에 wiki-link 추가
5. 사용자에게 생성 결과 + 연결된 노트 안내
```

### 노트 형식

| 입력 유형 | 생성 위치 | 템플릿 | 핵심 필드 |
|----------|----------|--------|----------|
| 아이디어 | `ideas/` | `idea.md` | `stage: seed` |
| 읽은 내용 | `workspace/notes/literature/` | `note.md` | `source` 필수 |
| 빠른 메모 | `inbox/` | (최소 frontmatter) | `type: fleeting` |
| 코드 조각 | `workspace/code/snippets/` | `snippet.md` | `language` |
| 오늘 배운 것 | `workspace/code/til/` | `til.md` | — |

## 2. 지식 연결 Agent

**트리거**: 사용자가 특정 주제에 대해 물어보거나, 노트 간 연결 요청 시

### 워크플로우

```
1. 주제 키워드 추출
2. frontmatter tags → 파일명 → 본문 순서로 관련 노트 검색
3. 연결 후보 제시 (최소 근거 1개: 공통 태그, 유사 주제, 시간적 근접)
4. 사용자 승인 시 양방향 wiki-link 추가
5. 3개 이상 연결된 클러스터 → MOC 생성 제안
```

### 검색 우선순위

1. `tags` 일치 (가장 정확)
2. `title` 유사도
3. `related` 필드의 기존 연결 그래프 탐색
4. 본문 키워드 매칭 (최후 수단)

---

## 3. 코드 개발 보조 Agent

**트리거**: 코드 스니펫 저장, TIL 작성, 기술적 문제 해결 요청 시

### 워크플로우

```
1. 코드/기술 내용 분류: snippet vs til vs 프로젝트 노트
2. 해당 템플릿 기반으로 노트 생성
3. language 필드 자동 감지 및 설정
4. 관련 snippet/til 검색 → 중복 확인
5. 기존 노트가 있으면 업데이트 제안, 없으면 새로 생성
```

---

## 4. 질문 & 검색 Agent

**트리거**: 사용자가 "~에 대해 알려줘", "~를 찾아줘" 라고 물을 때

### 워크플로우

```
1. 질문 의도 파악 (검색 vs 요약 vs 분석)
2. 검색 우선순위에 따라 관련 노트 탐색
3. 결과를 구조화하여 제시:
   - 직접 관련 노트 (제목 + 핵심 문장)
   - 간접 관련 노트 (연결 근거 설명)
   - 노트에 없는 내용이면 솔직하게 "기록된 내용이 없습니다" 안내
4. 필요시 새 노트 생성 제안
```

### 검색 우선순위

1. `tags` 일치
2. `title` 키워드
3. `related` 그래프 탐색
4. 본문 전문 검색
